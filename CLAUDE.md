# 언커버링 (Uncovering)

7일 진로 탐색 MVP 모바일 앱. 미션 수행 + 운영자 직접 분석 기반의 진로 성향 파악 서비스.

---

## 프로젝트 구성

본 디렉토리에는 두 가지가 존재 (또는 존재 예정):

1. **웹 데모 (현재 상태)**
   - Next.js 14 (App Router) + Tailwind CSS
   - 4페이지 플로우: 온보딩 → 미션 → 분석중 → 결과
   - API 라우트 목업 상태 (`/api/analyze`, `/api/analyze-partial`)
   - localStorage 키: `uncovering_missions`, `uncovering_result`

2. **MVP 모바일 앱 (개발 예정)**
   - 별도 Expo 프로젝트로 신규 구성
   - 본 문서 하단의 사양 참고

## MVP 모바일 앱 사양

### 컨셉
7일 진로 탐색 프로그램. 매일 3개 미션 → 자유서술 응답 → 운영자가 직접 분석 → 다음날 리포트 + 새 미션 전달.

### 기술 스택
| 영역 | 선택 |
|------|------|
| 프론트엔드 | Expo (React Native) + Expo Router + NativeWind |
| 백엔드 | Supabase (DB + Auth + Cron + Storage) |
| 푸시 알림 | Expo Notifications |
| 운영자 알림 | Resend (이메일) |
| 운영자 작업 도구 | 노션 or 구글시트 (Supabase 자동 적재) |
| 빌드 | EAS Build |
| 배포 | TestFlight 링크 (iOS) + APK 링크 (Android) — 앱스토어 등록 X |

### 운영 규칙
- 운영 기간: **7일**
- 일일 미션: **3개** (오전 10시 발송, 17시 마감, 엄격)
- 사용자당 총 응답: **21개**
- 행동축: **12개** (설계 진행 중)
- 미션풀: **50~60개** (설계 진행 중)
- 채점: 운영자 **수동** (Claude API 미사용)

### UX 결정사항
- **로그인**: 익명 + 닉네임 (UUID 기반, 비번/이메일 없음)
- **푸시 미수신 시**: 백업 없음 (사용자 책임)
- **마감 정책**: 17시 후 응답 불가, 스킵 처리
- **리포트 전달**: 앱 내 화면 (이미지/카드 형태)
- **베타 규모**: 최소 10명+ (미확정)
- **개인정보처리방침**: MVP 단계 생략

### 데이터 흐름
```
[사용자 폰]
  └ 매일 10시 푸시 알림
  └ 미션 화면 → 자유서술 응답 → "분석 중" 안내
        ↓
[Supabase]
  └ 응답 저장
  └ Edge Function → 운영자 이메일 발송 (Resend)
  └ 노션/구글시트 자동 적재
        ↓
[운영자]
  └ 시트에서 응답 확인 → 수동 채점 + 리포트 작성
  └ Supabase에 리포트 업로드
        ↓
[다음날 10시 Cron]
  └ 어제 리포트 + 새 미션 3개 푸시
```

### 알고리즘 운영 방식
- **IRT**: 사전 정의 파라미터(난이도 b, 변별도 a) 고정, theta만 일일 업데이트
- **Contextual Bandit**: Day 2부터 작동 (Day 1은 행동축 커버리지 우선)
- 미션 파라미터 재추정: 누적 사용자 100명+ 이후 EM 알고리즘

## 미션풀 설계 가이드 (팀원 작업 중)

각 미션이 가져야 할 메타데이터:
- `mission_id`, `scenario`, `prompt`
- `trait_axes`: 측정 행동축 3~4개
- `difficulty (b)`: -3 ~ +3
- `discrimination (a)`: 0.5 ~ 2.5 (1.0~1.7 권장)
- `axis_weights`: 축별 측정 비중 (합=1.0)
- `expected_time_min`: 응답 예상 시간 (3~5분 목표)

난이도 분포 (60개 기준): 쉬움 20% / 중간 60% / 어려움 20%
일일 3미션 구성: 아침(쉬움~중간) → 점심(중간~어려움) → 저녁(중간)

## 설계 결정 기록

- **운영 기간**: 7일 → 3일 테스트 먼저 진행 후 확장
- **일차 전환**: 당일 17시까지 3미션 완료 → "전부 완료했어요!" 클릭 → 다음날 10시까지 대기
- **푸시 알림(B안)**: 베타 직전에 추가 예정. 테스트 단계는 앱 열 때 시각 체크(A안)로 대체
- **개발 클라이언트**: expo-dev-client 빌드 사용, 이후 JS 수정은 `npx expo start`로 즉시 확인
- **리포트 생성**: 운영자 수동 작성 (Claude API 미사용). 운영자가 노션 리포트 DB에 직접 입력 → Supabase Edge Function이 읽어 앱에 표시
- **운영자 워크플로우**: 노션 응답 수신 DB에서 미션+답변 확인 → 노션 리포트 DB에 행동축/점수/설명 입력 → 사용자가 앱 열면 자동 표시
- **노션 리포트 DB 구조 (B안)**: 닉네임, 일차, 요약, 축이름/점수/설명 × 4개, 다음힌트
